## Installation

Vendoring is the **production model** (`source()` the helpers from a
consumer repo's `00_functions/`). Native R-package installation is
supported for development and CI only.

### Option A — vendor into a consumer repo (production)

Drop the helpers you need into the consumer's `00_functions/` and pin
the version in `.toolkit_manifest.yml`:

```r
source(file.path(rootFolder, "00_functions", "dw_io.R"))
source(file.path(rootFolder, "00_functions", "dw_api.R"))
source(file.path(rootFolder, "00_functions", "cso_toolkit_sync.R"))
```

```yaml
# 00_functions/.toolkit_manifest.yml
source: "unicef-drp/cso-toolkit"
pulled_version: "v0.2.0"
pulled_date: "2026-05-25"
```

See [`templates/.toolkit_manifest.yml`](../templates/.toolkit_manifest.yml)
for the full schema and [`R/README.md`](R/README.md) for the per-helper
catalogue.

### Option B — install as an R package (development)

```r
# from the repo root
devtools::install_local("r/")
library(csotoolkit)
```

This is what `R CMD check` and the pkgdown site build use; vendoring
consumers do NOT need it.

## Quick start

```r
# 1. Source the helpers
source("r/R/dw_io.R")
source("r/R/dw_api.R")

# 2. Set session-level globals (normally done by profile_<repo>.R)
teamsWrkData <- "/path/to/wrk"
teamsRawData <- "/path/to/raw"
dw_mode <- "producer"
dw_apis_allowed <- TRUE

# 3. Use the contract
library(dplyr)
df <- tibble(REF_AREA = c("AGO", "BFA"), OBS_VALUE = c(0.5, 0.7))
out_path <- dw_save(
  df,
  name = "dw_ed_edu.csv", sector = "ed", kind = "wrk",
  isid = "REF_AREA",
  metadata = list(
    title = "Education indicators",
    producer = "01_dw_prep/012_codes/ed/example.R",
    sources = c("UIS bulk SDG_092025"),
    vintage = "2026-05"
  )
)
# Writes the CSV + a sibling `.provenance.json` sidecar.

# 4. Read back
warehouse <- dw_use(name = "dw_ed_edu.csv", sector = "ed", kind = "wrk")
```

## Mode contract (R side)

`dw_save` raises `stop()` when reviewer-session writes would land under
canonical, unless `allow_canonical_write = TRUE` is passed. `dw_api_fetch`
similarly refuses live API calls in reviewer sessions and reads the
cached deposit instead. The full contract is documented in
[`docs/mode_contract_integration.md`](../docs/mode_contract_integration.md)
and [`docs/roles_and_workflow.md`](../docs/roles_and_workflow.md).

## Error envelope

Every `stop()` / `warning()` in the package emits the three-part
**WHAT / Why / Fix** envelope:

```text
[cso_toolkit.dw_save] Reviewer mode forbids writes under canonical: /path
  Reviewer sessions must keep canonical deposits read-only to preserve
  vintage permanence; writes go to the sandbox.
  Fix:
    1. Resolve a sandbox path instead, OR
    2. If this is a deliberate DBM bootstrap, pass `allow_canonical_write = TRUE`.
```

The leading `[cso_toolkit.<func>]` prefix is grep-friendly — search the
project for sites that hit a given error class.
